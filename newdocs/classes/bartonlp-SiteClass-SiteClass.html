getLastInsertId()

dbPdo.class.php : 445

Get the ID from the last insert

public getLastInsertId() : int

WARNING NEVER do multiple inserts with AUTO_INCREMENT without doing this method in between. If we need to do 'insert ... on duplicate key' we better not need the insert id. If we do we should do an insert in a try block and an update in a catch. That way if the insert succeeds we can do the getLastInsertId() after the insert. If the insert fails for a duplicate key we do the update in the catch. And if we need the id we can do a select to get it (somehow). Note if the insert fails because we did a 'insert ignore ...' then last_id is zero and we return zero.

Return values
int —

Last insert id

Execute a Sql statment

public sql(string $query[, array<string|int, mixed> $params = [] ]) : int|bool

Execute a single SQL statement with optional prepared parameters. Supports SELECT, INSERT, UPDATE, DELETE (DML), as well as CREATE, DROP, ALTER, TRUNCATE (DDL), and GRANT, REVOKE, SET, USE (DCL). Automatically uses prepare/execute if parameters are supplied.

Parameters

$params : array<string|int, mixed> = []

Optional array of values to bind to the statement (for prepared execution).

Tags
throws
Exception
  • On SQL preparation or execution failure.
site-effect
  • For SELECT/SHOW/EXPLAIN sets $this->result. NOTE: $query must be a single line with no newlines.
Return values
int|bool —
  • Row count for INSERT, UPDATE, DELETE
  • True for successful DDL/DCL statements
  • Row count for SELECT/SHOW/EXPLAIN (and sets $this->result)
  • 0 for suppressed tracking query failures
  • false only if something unusual occurs

sqlPrepare()

dbPdo.class.php : 282

Does a SQL prepare

public sqlPrepare(string $query[, array<string|int, mixed> $values = null ]) : bool|int

BLP 2025-05-07 - OLD LOGIC This method works with fully formed queries! That is no :name or =? that need to be prepared! It also seems to work with =|<|>|<=|>=|!=|like. It also seems to work with 'between'. It only worked with ':named' parameters but not with '?' parameters. This method can be used with $this->fetchrow(..).

If you have a complicated queries or '?' parameters use the RAW PDO prepare(), bind_params(), execute(), bind_result() and fetch(). Used as follows with bound params:

  1. $username="bob"; $query = "select one, two from test where name=?";
  2. $stm = PDO::prepare($query);
  3. $stm->bind_param("s", $username);
  4. $stm->execute();
  5. $stm->bind_result($one, $two);
  6. $stm->fetch();
  7. echo "one=$one, two=$two
    "; NOTE: we do not have a bind_param(), execute(), bind_result() or fetch() functions in this module. You will have to use the native PHP functions with the returned $stm. NOTE: As of PHP 8 PDO uses exception as the default: PDO::ERRMODE_EXCEPTION.
